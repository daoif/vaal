请为以下模块生成测试任务和实现任务。

## 输出格式

```
- [ ] [TEST-{编号}] {测试任务描述}
  
  **关联模块:** {{MODULE_ID}}
  
  **硬约束:**
  - {约束内容}

- [ ] [IMPL-{编号}] {实现任务描述}
  
  **关联模块:** {{MODULE_ID}}
  
  **硬约束:**
  - {约束内容}
  
  **依赖:**
  - {前置任务ID}
```

## 生成原则

1. **测试优先** - 每个功能先写测试任务
2. **粒度适中** - 每个任务应该能在一次 AI 会话中完成（约 30 分钟工作量）
3. **依赖明确** - 实现任务依赖对应的测试任务
4. **约束继承** - 任务约束应与模块约束一致

## 任务 ID 规则

- 任务 ID 必须全局唯一，使用 4 位数字编号：`MMTT`
  - `MM` = 模块序号（两位，不足补 0）
  - `TT` = 任务序号（两位，从 01 开始，不足补 0）
- 测试任务: `TEST-MMTT`，例如：
  - MOD-001 的第 1 个任务：`TEST-0101`
  - MOD-010 的第 1 个任务：`TEST-1001`
- 实现任务: `IMPL-MMTT`，例如：
  - MOD-001 的第 1 个任务：`IMPL-0101`
  - MOD-010 的第 1 个任务：`IMPL-1001`
- 当前模块序号（两位）: {{MODULE_INDEX}}
