请为以下模块生成测试任务和实现任务。

## 输出格式

```
- [ ] [TEST-{编号}] {测试任务描述}
  
  **关联模块:** {{MODULE_ID}}
  
  **硬约束:**
  - {约束内容}

- [ ] [IMPL-{编号}] {实现任务描述}
  
  **关联模块:** {{MODULE_ID}}
  
  **硬约束:**
  - {约束内容}
  
  **依赖:**
  - {前置任务ID}
```

## 生成原则

1. **测试优先** - 每个功能先写测试任务
2. **粒度适中** - 每个任务应该能在一次 AI 会话中完成（约 30 分钟工作量）
3. **依赖明确** - 见下方依赖规则
4. **约束继承** - 任务约束应与模块约束一致

## 依赖规则

支持两种依赖格式：

1. **任务依赖**：`TEST-0001-0001`、`IMPL-0001-0001`
   - 表示依赖某个具体任务完成

2. **模块依赖**：`MOD-0001`（四位数字）
   - 表示依赖整个模块完成（该模块的所有任务都完成）
   - 用于跨模块依赖

### 使用场景

- **模块内依赖**：用具体任务 ID（如 `IMPL-0001-0001`）
- **跨模块依赖**：用模块 ID（如 `MOD-0001`）

### 模块内依赖

- 每个 IMPL 任务**必须**依赖它对应的 TEST 任务
- 如果 IMPL 的实现需要**调用**其他 IMPL 实现的函数/能力，**也必须**依赖那些 IMPL

示例：`IMPL-0001-0006`（统一校验入口）需要调用 `IMPL-0001-0001`（结构校验）的函数，则：
```
**依赖:**
- TEST-0001-0006      ← 自己的测试
- IMPL-0001-0001      ← 要调用的函数所在任务
- IMPL-0001-0002
- IMPL-0001-0003
```

## 任务 ID 规则

- 任务 ID 必须全局唯一，使用分隔符方案：`MMMM-TTTT`
  - `MMMM` = 模块序号（四位，不足补 0）
  - `TTTT` = 任务序号（四位，从 0001 开始，不足补 0）
- 测试任务: `TEST-MMMM-TTTT`，例如：
  - MOD-001 的第 1 个任务：`TEST-0001-0001`
  - MOD-010 的第 1 个任务：`TEST-0010-0001`
  - MOD-100 的第 100 个任务：`TEST-0100-0100`
- 实现任务: `IMPL-MMMM-TTTT`，例如：
  - MOD-001 的第 1 个任务：`IMPL-0001-0001`
  - MOD-010 的第 1 个任务：`IMPL-0010-0001`
  - MOD-100 的第 100 个任务：`IMPL-0100-0100`
- 当前模块序号（四位）: {{MODULE_INDEX}}
