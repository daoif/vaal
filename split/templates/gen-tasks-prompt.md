## 输出格式

```
- [ ] [TEST-{编号}] {测试任务描述}
  
  **关联模块:** {{MODULE_ID}}
  
  **硬约束:**
  - {约束内容}

- [ ] [IMPL-{编号}] {实现任务描述}
  
  **关联模块:** {{MODULE_ID}}
  
  **硬约束:**
  - {约束内容}
  
  **依赖:**
  - {前置任务ID}
```

## 生成原则

1. **测试优先** - 每个功能先写测试任务
2. **粒度适中** - 每个任务应该能在一次 AI 会话中完成（约 30 分钟工作量）
3. **依赖明确** - 实现任务依赖对应的测试任务
4. **约束继承** - 任务约束应与模块约束一致

## 任务 ID 规则

- 测试任务: TEST-{模块序号}{任务序号}，如 TEST-101、TEST-102
- 实现任务: IMPL-{模块序号}{任务序号}，如 IMPL-101、IMPL-102
- 当前模块序号: {{MODULE_INDEX}}
